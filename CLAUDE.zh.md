## 仓库结构
- `templates/zh/` - 中文简历分赛道模板
- `.claude/skills/job-application-assistant/` - 工作流参考文件（含 `08-resume-zh.md`、`09-da-zhaohu-zh.md`）

- `cv/`、`cover_letters/` - 原版 LaTeX 英文模板（海外/英文岗用）

## 国内岗位工作流
1. 用户提供岗位（岗位链接或文本）
2. **先评估匹配**：技能匹配、经历匹配、文化匹配，呈现评估后再继续
3. 若匹配：生成中文简历（`documents/zh/resume_<公司>.md`）+ 打招呼话术/求职信（`documents/zh/`）
4. **校验两份内容**（见下方核查清单）
5. 基于岗位要求与个人优势准备面试要点

**重要**：所有生成内容必须真实，不得虚构技能/经历/业绩；缺口诚实表述。

## 核查清单（生成简历/话术后必做）

### 事实准确
- [ ] 所有陈述与真实画像一致 — 无虚构技能、经历、业绩
- [ ] 公司名、岗位、时间、地点正确
- [ ] 联系方式正确

### 针对性
- [ ] 个人总结/开场针对具体岗位（非套话）
- [ ] 技能与经历要点重构以匹配岗位要求
- [ ] 岗位关键要求均已覆盖（缺口诚实标注）

### 一致性
- [ ] 中文简历符合 `08-resume-zh.md` 的一页纸/分赛道规则
- [ ] 打招呼话术点名公司与岗位，无群发感
- [ ] 简历与话术口径一致，无矛盾

### 质量
- [ ] 无错别字、无语法错误
- [ ] 量化结果优先（"提升 X%" 优于形容词）
- [ ] 投递前导出 .docx（WPS/Word）或 .pdf

### 合规
- [ ] 未虚构内容
- [ ] 敏感个人信息本地处理，谨慎分享（PIPL）
- [ ] 由用户手动在 App 内投递，未自动投递
